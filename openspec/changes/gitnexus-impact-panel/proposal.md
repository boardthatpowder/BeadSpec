## Why

The repo CLAUDE.md and the `gitnexus-impact-reminder.sh` hook mandate running GitNexus impact analysis before editing any symbol, but every `mcp__gitnexus__impact` / `context` call today lives in the agent transcript and disappears. The user has zero persistent UI surface for "what breaks if I touch this." The Tier-1 ★ item in the workflow-vs-UI gap review (`review-the-updates-that-shimmering-volcano.md`, cluster 1) ranked this as the highest-leverage GitNexus surface precisely because the signal is already being produced — the hook already nags, `getGitRefsForIssue` already returns per-issue diff strings, and the `TaskDetailPanel` tab pattern (`Details / Dependencies / Activity`) is the natural home.

Issues already carry the seeding data: the issue's branch diff names the symbols that were changed, and `getGitRefsForIssue` already returns that diff for the existing Activity tab badge. The only missing piece is a command that shells out to `npx gitnexus impact --target <symbol> --json` and a tab that renders the result.

## What Changes

- `TaskDetailPanel` SHALL render a new `Impact` tab as the fourth tab, after `Activity`.
- The Impact tab SHALL present candidate symbol names extracted from the issue's diff (`GitRefs.diff`) as clickable chips, sorted by occurrence count, capped at 10. A free-text fallback input SHALL always be visible.
- Selecting a candidate or submitting the manual input SHALL invoke a new Tauri command `run_gitnexus_impact(project_path, symbol)` that shells out to `npx gitnexus impact --target <symbol> --json` with a 15-second timeout.
- The command returns a `GitnexusImpactReport` containing: `symbol`, `risk` (`Low | Medium | High | Critical | Unknown`), `upstream_by_process` (callers grouped by GitNexus process), `downstream` (flat callee list), `affected_processes` (slug list), and `index_status` (`Fresh | Stale | Unknown`).
- The panel SHALL render a risk badge using the project's existing `LABEL_CHIP_COLORS` palette, grouped upstream callers, downstream callees, and an affected-processes count.
- Results SHALL NOT be cached across panel mounts. A `Refresh` button SHALL re-run the most recent call.
- All failure modes SHALL be explicit inline callouts: CLI not found, no index, stale index, symbol not found, and timeout.

Non-goals (explicit):
- No result persistence — neither Dolt nor Ruflo memory.
- No write path — no "Re-analyze index" button.
- No multi-symbol batch analysis.
- No graph visualisation — flat grouped lists only.
- No MCP-direct calls from the renderer — only `tauri-specta`-generated IPC.
- No background polling.
- No coupling to the OpenSpec change browser.

## Capabilities

### New Capabilities
- `gitnexus-impact-panel`: owns the `ImpactPanel` component, symbol-candidate extraction from diff, the `run_gitnexus_impact` IPC command contract, risk-badge palette mapping, and all empty / error states.

### Modified Capabilities
- `task-detail`: adds an `Impact` tab to the fixed inner-sub-tab set and extends the per-`paneId:taskId` persistence scenario to include the new tab id.

## Impact

- **Tauri command** (new): `run_gitnexus_impact` in `src-tauri/src/commands/gitnexus.rs`. Reuses the shell-resolved-PATH helper from `commands::external`. New module registered in `src-tauri/src/commands/mod.rs`; command added to both `tauri::generate_handler!` and `tauri_specta::collect_commands!` in `src-tauri/src/lib.rs`.
- **Type bindings**: `GitnexusRisk`, `GitnexusCaller`, `GitnexusProcessGroup`, `GitnexusIndexStatus`, `GitnexusImpactReport` auto-generated by `specta` into `src/bindings.ts`.
- **Frontend IPC**: `src/ipc.ts` gains `runGitnexusImpact(projectPath, symbol)`.
- **React component** (new): `src/components/task-detail/ImpactPanel.tsx`.
- **TaskDetailPanel diff**: extend `TabId` union, append `{ id: "impact", label: "Impact" }` to tabs array, render `<ImpactPanel>` for `activeTab === "impact"`.
- **Symbol extraction**: `extractDiffSymbols(diff: string): string[]` pure helper in `src/components/task-detail/`, unit-tested.
- **No Dolt schema changes.** No new Beads tables or write paths.
